Description from the author:

"SREhttp/2 is an http/1.1 compliant World Wide Web server for OS/2. More precisely, SREhttp/2 is a "filter" for the SRE2003 Internet Interface

Basically, SRE2003 provides and "API" that facilitates working with http requests. SREhttp/2 uses this "API" to implement an http/1.1 web server.

Other filters exist for SRE2003, including a Simple filter that comes packaged with SRE2003. For those who do not need the rich feature set of SREhttp/2, use of this Simple filter (or the SRELite/2 filter) may be advantageous."

This packages contains the REXX DLL UNZIPAPI.DLL. There is a note about the documentation for this DLL in the packae:

"15 Apr 1998 (updated 5 Sept 2002)

This [package] contains an error-ridden description of UNZIPAPI.DLL. I've found that in some places where a "stem" variable is claimed to be needed, you have to use a string instead. And unziptovar chokes when a file name contains a &."

Please note that I did not check if this is right or wrong! And I don't know if this is also true for the Unzip REXX DLL Documentation included in REXX Tips & Tricks.

Also included in this package is a DLL called rxcpage.dll but there is no documentation for this DLL.
