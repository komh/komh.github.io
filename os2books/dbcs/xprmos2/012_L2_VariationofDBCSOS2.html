Variation of DBCS OS/2

Though there is only one version of SBCS OS/2, DBCS OS/2 has several variants according to the supported language. The following DBCS NLVs are available from IBM.

  • OS/2 H (Hanguel OS/2)

    It supports DBCS functions for Hanguel (Korean) such as Hanguel fonts and a Hanguel input method. The supported code page has been changed to Korean Standard code (KS code, CP 949) from IBM Standard code (CP 934/944) on the version 2.00.1 and later. MVDM environment supports PC-DOS, DOS H/V, WIN-OS2 and WIN-OS2 H.

  • OS/2 J (Japanese OS/2)

    It supports DBCS functions for Japanese such as Japanese fonts and a Japanese input method. The ATM outline fonts and True Type fonts are available in addition to the short vector outline fonts. MVDM environment supports PC-DOS, DOS J/V, WIN-OS2 J.

  • OS/2 P (S-Chinese OS/2)

    S-Chinese means Simplified Chinese, which uses ideographs with simplified shape and used mainly in P.R.C. (People's Republic of China). OS/2 P supports DBCS functions for Simplified Chinese such as Simplified Chinese fonts and a Chinese input method. The OS/2 P is available from Version 2.1 and supports combined code page 1381, known as GB code. MVDM environment supports PC-DOS, WIN-OS2 and WON-OS2 P.

  • OS/2 T (R.O.C. OS/2)

    R.O.C. is an abbreviation for the Republic Of China. Traditional Chinese, which uses ideographs with traditional shapes, is used in R.O.C.. OS/2 T supports DBCS functions for Traditional Chinese such as Traditional Chinese fonts and a Chinese input method. MVDM environment supports PC-DOS, DOS T/V, WIN-OS2 and WIN-OS2 T.

    Though there were code page features on OS/2 T2.1, OS/2 Warp T supports Big 5 code (CP 950) only.
