SEND KEY (3)

SEND KEY data is not affected by the current shift or CAPS lock status. This is different from the SBCS version.

In DBCS 3270 Emulator:

  • SO/SI are automatically inserted at required positions when SBCS/DBCS mixed strings are allowed.

  • The current keyboard SBCS/DBCS mode does not affect SEND KEY.

In DBCS 5250 Emulator, SO/SI are not inserted automatically for DBCS Either Field. You should send SO/SI Creation ASCII Mnemonic (@X@5) at first to enter DBCS in this field.

Note: Hitting SO/SI Creation key of DBCS 5250 creates a pair of SO and SI and turns the insert mode on so that the following input is inserted between the SO and SI. If you use the AUTORESET session option, the insert mode set by SO/SI Creation key is also reset and you cannot continue the input. Use NORESET option.

The following are DBCS EHLLAPI unique ASCII Mnemonics. You may need to use SO/SI Creation (as explained above) and Vertical Line, Broken (as explained in the note below). 

                                3270  5250
@X@1 ... SO/SI Display            Y     Y
@X@5 ... SO/SI Creation           N     Y
@X@6 ... Attribute Display        N     Y
@X@7 ... Character Advance        N     Y
@X@c ... Vertical Line, Broken*   N     Y

NOTE: *) This character exists in host code page 037 which is supported
         by Japanese 5250 Emulator (code page 037) but not in Japanese
         PC code pages.
