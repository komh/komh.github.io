Run-Time Dynamic Link of DBCS OS/2 Unique Functions

As explained in the previous sections, there are some API calls that only exists in DBCS OS/2.

  • WinDBCSModeControl

See MDI (Multiple-Document Interface) for relevant information on the above-named functions.

TRNSCALL.DLL includes TrnsDt. See TrnsDt for more information on this function.

Using the DBCS OS/2 unique API calls is not recommended since they are not supported by the worldwide OS/2, SBCS OS/2. If you must use them, use run-time dynamic linking (not load-time dynamic linking) and call them only when they exist.
