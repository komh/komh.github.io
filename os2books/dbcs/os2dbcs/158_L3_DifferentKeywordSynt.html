Different Keyword/Syntax

There are differences in a parameter keyword and syntax of DBCS 3270 File Transfer.

    JISCII

      Japanese 3270 File Transfer uses this keyword instead of ASCII. (Korean and T-Chinese 3270 File Transfers use ASCII as does SBCS 3270 File Transfer.)
    [
      A left bracket has to be specified just after the host file name followed by one or more blanks, for example: 
      REM ----------- For sending a Japanese text file to VM------------
send c:report.dat a2:report script a [(CRLF JISCII
REM ----------- For sending a Korean/Chinese text file to VM------
send c:report.dat a2:report script a [(CRLF ASCII
