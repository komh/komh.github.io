Horizontal Cursor Move

The table in Horizontal Cursor Move shows the number of display positions to move for horizontal character cursor positioning. 

┌────────────┬─────────────────────┬─────────────────────┐
│            │    Cursor Left      │    Cursor Right     │
│ Cursor is  │ Next        Cursor  │ Next        Cursor  │
│ positioned │ character   moves   │ character   moves   │
│ on:        │ to left:    left:   │ to right:   right:  │
├────────────┼───────────┬─────────┼───────────┬─────────┤
│ SBCS       │ SBCS      │ 1       │ SBCS      │ 1       │
│            ├───────────┼─────────┼───────────┼─────────┤
│            │ DBCS      │ 2       │ DBCS      │ 1       │
├────────────┼───────────┼─────────┼───────────┼─────────┤
│ DBCS       │ SBCS      │ 1       │ SBCS      │ 2       │
│            ├───────────┼─────────┼───────────┼─────────┤
│            │ DBCS      │ 2       │ DBCS      │ 2       │
└────────────┴───────────┴─────────┴───────────┴─────────┘
Horizontal Cursor Move

In short,

  • If the right arrow key is pushed and the current character is DBCS, you should move the cursor two display positions to the right.

  • If the left arrow key is pushed and the next character on the left is DBCS, you should move the cursor two display positions to the left.

    For example, 

    aD1D2b <-> aD1D2b <-> aD1D2b <-> aD1D2b
-           -            -            -
