Functional Verification Test

This testing is to ensure that the DBCS enabled functions are implemented correctly in the software product. The following items are required to be tested carefully:

  • The input DBCS data are handled correctly.

    • Change the DBCS keyboard and input method mode.

    • Input DBCS characters from the keyboard.

    • Use the Cut-and-Paste function to input the DBCS text.
    
      The output DBCS data are displayed correctly.

      • Try to display the DBCS data.

        The READMEJ file in the root directory of the system drive is a good source of DBCS data.

      • Change font option

        Use both SBCS and DBCS font to display the data.

      • Test in different output devices

        Try different output devices, normal and high resolution display devices. Try the printer device.

      
        The connections with other system are handled correctly.

        If the software products can connect and communicate with other system, for example a client server application, then you need to test the connection and data transfer handling. You need to test the data conversion, make sure that the DBCS data are converted and transferred correctly.

      
        The error cases for unsupported DBCS data are handled correctly.

        If the software product has limitation on DBCS handling, you need to make sure that the error cases are handled correctly. If the software product provides SBCS/English PII or MRI, you need to make sure that they are displayed correctly in the DBCS system. Be aware that some SBCS characters are not supported in the DBCS code page.

      The following considerations are also required:

      • The NLVs of the target base system softwares are available.

      • The related NLVs of the required softwares are available.

      • The PS/2 hardwares can be applied as the substitute of DBCS hardwares, if the DBCS OS/2 supports these hardwares.
