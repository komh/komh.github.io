EnumFonts (Modify)

EnumFonts

    Functional Difference from WIN95

    • The Open32 NLS Extension will also return facename with " OEM" facename qualifier, when an OS/2 UGL font can support the system OEM codepage. This allows users to operate with text which was written in OS/2 codepages.
    Functional Difference from SBCS Open32

    • The CharSet and PitchAndFamily member value will contain appropraite values.
    • More than one facename will be returned for one OS/2 UGL font. In this case, the facename qualifiers may be supplied to the end of the facenames for users to distinguish one from the others, as Windows does.
