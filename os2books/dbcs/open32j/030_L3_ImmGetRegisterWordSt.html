ImmGetRegisterWordStyle

ImmGetRegisterWordStyle

    Functional Difference from WIN95

    IME_REGWORD_STYLE_EUDC is not supported.

    Functional Difference from SBCS Open32

    'N/A'

    Implementation

    This API is translated to ImQueryRegisterWordType(HIMI, ulItem, pWordType, pulCount). And using HKL of WIn95 , an appropriate IME of OS/2 is selected.

    The STYLEBUF of Win95 is translated to the WORDTYPE of OS/2.

    The WORDTYPE.ulStyle is T.B.D.

    Behavior using 16 bit IME

    Not supported.
