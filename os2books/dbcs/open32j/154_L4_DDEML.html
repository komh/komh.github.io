DDEML

DDEML (Dynamic Data Exchange Management Library) provides functions to application programs to exchange connection-based, DDE messages. Strings used for service name, topic name, and item name should be created with CP_WINANSI codepage ID on Windows 95, though Windows NT supports CP_WINUNICODE in addition (*1). (CP_WINANSI represents the system default ANSI codepage.) This means that no code conversion will take place for those names on Windows 95. On the other hand, DDE object format is the same as standard clipboard format, and so CF_LOCALE support will be necessary as same as the enhancement for Clipboard.

Note:

  • The WIN32 spec reads that only CP_WINANSI (1004) is accepted on Windows 95 but, in reality, the system default ANSI codepage is also accepted. For example, Window 95, Japanese version accepts 1004(CP_WINANSI) and 932(Japanese ANSI codepage).
