Variation of DBCS OS/2

Though there is only one version of SBCS OS/2, DBCS OS/2 has several variants according to the supported language. The following DBCS NLVs are available from IBM.

  • OS/2 H (Hanguel OS/2)

    It supports DBCS functions for Hanguel (Korean) such as Hanguel fonts and Hanguel input method. The supported code page has been changed to Korean Standard code (KS code, CP 949) from IBM Standard code (CP 934/944) on the version 2.00.1 and later. MVDM environment supports PC-DOS, DOS H/V and SBCS Windows. The PS/55 display adapter is not supported any more since H2.00.1.

  • OS/2 J (Japanese OS/2)

    It supports DBCS functions for Japanese such like Japanese fonts and Japanese input method. The ATM outline fonts are available other than the short vector outline fonts. MVDM environment supports PC-DOS, DOS J/V, DOS J, SBCS Windows and Japanese Windows.

    
      OS/2 P (P.R.C. OS/2)

      P.R.C. is an abbreviation for the People's Republic of China. Simplified Chinese, which uses ideograph with simplified shape, is used in P.R.C.. OS/2 P supports DBCS functions for Simplified Chinese such as Simplified Chinese fonts and Chinese input method. The OS/2 P is available from V2.1 and supports combined code page 1381, known as GB code.MVDM environment supports PC-DOS, DOS P/V and SBCS Windows.

    
      OS/2 T (R.O.C. OS/2)

      R.O.C. is an abbreviation for the Republic Of China. Traditional Chinese, which uses ideograph with traditional shape, is used in R.O.C.. OS/2 T supports DBCS functions for Traditional Chinese such as Traditional Chinese fonts and Chinese input method. MVDM environment supports PC-DOS, DOS T/V, DOS T and SBCS Windows.

    There are a feature for supported code page on Version 2.1. That is, there are two version of OS/2 T2.1, the one supports Big 5 code (CP 950) and the other supports IBM Standard code (CP 938/948).
