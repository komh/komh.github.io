Adding New Command Messages

Following are four steps a media driver author needs to follow to add support for a device-specific message to a driver:

  • Define the new or modified media control message your driver plans to support. (See Defining New Messages.)

  • Define the new data structure and flags for this message. (See Defining New Data Structures and Flags.)

  • Create a custom command table that MDM can use to parse the command string in mciSendString, so it can create the appropriate data structure to pass to your driver. (See Parsing a Command List).

  • Support the new message in your MCD's entry point (mciDriverEntry).
