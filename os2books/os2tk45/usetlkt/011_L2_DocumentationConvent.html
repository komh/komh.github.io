Documentation Conventions

Throughout Using Your Toolkit, the following conventions distinguish the different elements of text:

    plain text

      Function names, structure names, data type names, message names, enumerated types, and constant names.
    Initial capitalization
      Key names, group-box controls, drop-down list boxes, dialog windows, combo-boxes, single-line entry (SLE) and multiple-line entry (MLE) fields.
    UPPERCASE
      File names and error codes.
    monospace
      Programming examples and user input at the command line prompt or into an entry field.
    bold
      Push buttons, check boxes, radio buttons, spin buttons, menu bar choices, and menu items.
    italics
      Parameters, structure fields, titles of documents, and first occurrences of words with special meaning.
