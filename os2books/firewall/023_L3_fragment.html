fragment

Fragment control.

    Syntax

      f=no | f=yes | f=only | f=headers
    Description
      Indicates how the rule should handle fragmented versus non-fragmented packets. The three types of traffic are non-fragments (complete packets), fragments, and fragment headers.

      If this parameter is not specified, the default behaviour is f=yes.
